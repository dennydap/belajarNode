<h3>
<a id="headings" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#headings" aria-hidden="true"><span class="octicon octicon-link"></span></a>Headings</h3>

<p>To create a heading, add one to six <code>#</code> symbols before your heading text. The number of <code>#</code> you use will determine the size of the heading.</p>

<pre><code># The largest heading
## The second largest heading
###### The smallest heading
</code></pre>

<p><img src="./readme_files/headings-rendered.png" alt="Rendered H1, H2, and H6 headings"></p>

<h3>
<a id="styling-text" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#styling-text" aria-hidden="true"><span class="octicon octicon-link"></span></a>Styling text</h3>

<p>You can indicate emphasis with bold, italic, or strikethrough text.</p>

<h3>
<a id="quoting-text" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#quoting-text" aria-hidden="true"><span class="octicon octicon-link"></span></a>Quoting text</h3>

<p>You can quote text with a <code>&gt;</code>.</p>

<pre><code>In the words of Abraham Lincoln:

&gt; Pardon my French
</code></pre>

<p><img src="./readme_files/quoted-text-rendered.png" alt="Rendered quoted text"></p>

<div class="alert tip">

<p><strong>Tip:</strong> When viewing an issue or pull request, you can automatically quote text in a reply by highlighting the text, then typing <code>r</code>. For more information, see "<a href="https://help.github.com/articles/using-keyboard-shortcuts/">Using keyboard shortcuts</a>."</p>

</div>

<h3>
<a id="quoting-code" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#quoting-code" aria-hidden="true"><span class="octicon octicon-link"></span></a>Quoting code</h3>

<p>You can call out code or a command within a sentence with single backticks. The text within the backticks will not be formatted.</p>

<p><code>Use `git status` to list all new or modified files that haven't yet been committed.</code></p>

<p><img src="./readme_files/inline-code-rendered.png" alt="Rendered inline code block"></p>

<p>To format code or text into its own distinct block, use triple backticks.</p>

<pre>Some basic Git commands are:
```
git status
git add
git commit
```
</pre>

<p><img src="./readme_files/code-block-rendered.png" alt="Rendered code block"></p>

<p>For more information, see "<a href="https://help.github.com/articles/creating-and-highlighting-code-blocks">Creating and highlighting code blocks</a>."</p>

<h3>
<a id="links" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#links" aria-hidden="true"><span class="octicon octicon-link"></span></a>Links</h3>

<p>You can create an inline link by wrapping link text in brackets <code>[ ]</code>, and then wrapping the URL in parentheses <code>( )</code>. You can also use the keyboard shortcut <code>command + k</code> to create a link.</p>

<p><code>This site was built using [GitHub Pages](https://pages.github.com/).</code></p>

<p><img src="./readme_files/link-rendered.png" alt="Rendered link"></p>

<div class="alert tip">

<p><strong>Tip:</strong> GitHub automatically creates links when valid URLs are written in a comment. For more information, see "<a href="https://help.github.com/articles/autolinked-references-and-urls">Autolinked references and URLS</a>."</p>

</div>

<h3>
<a id="section-links" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#section-links" aria-hidden="true"><span class="octicon octicon-link"></span></a>Section links</h3>

<p>You can link directly to a section in a rendered file by hovering over the section heading to expose the link:</p>

<p><img src="./readme_files/readme-links.png" alt="Section link within the README file for the github/scientist repository"></p>

<h3>
<a id="relative-links" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#relative-links" aria-hidden="true"><span class="octicon octicon-link"></span></a>Relative links</h3>

<p>You can define relative links and image paths in your rendered files to help readers navigate to other files in your repository.</p>

<p>A relative link is a link that is relative to the current file. For example, if you have a  README file in root of your repository, and you have another file in <em>docs/CONTRIBUTING.md</em>, the relative link to <em>CONTRIBUTING.md</em> in your README might look like this:</p>

<pre><code>[Contribution guidelines for this project](docs/CONTRIBUTING.md)
</code></pre>

<p>GitHub will automatically transform your relative link or image path based on whatever branch you're currently on, so that the link or path always works. You can use all relative link operands, such as <code>./</code> and <code>../</code>.</p>

<p>Relative links are easier for users who clone your repository. Absolute links may not work in clones of your repository - we recommend using relative links to refer to other files within your repository.</p>

<h3>
<a id="lists" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#lists" aria-hidden="true"><span class="octicon octicon-link"></span></a>Lists</h3>

<p>You can make a list by preceding one or more lines of text with <code>-</code> or <code>*</code>.</p>

<pre><code>- George Washington
- John Adams
- Thomas Jefferson
</code></pre>

<p><img src="./readme_files/unordered-list-rendered.png" alt="Rendered unordered list"></p>

<p>To order your list, precede each line with a number.</p>

<pre><code>1. James Madison
2. James Monroe
3. John Quincy Adams
</code></pre>

<p><img src="./readme_files/ordered-list-rendered.png" alt="Rendered ordered list"></p>

<p>You can create nested lists by indenting lines by two spaces.</p>

<pre><code>1. Make my changes
  1. Fix bug
  2. Improve formatting
    * Make the headings bigger
2. Push my commits to GitHub
3. Open a pull request
  * Describe my changes
  * Mention all the members of my team
    * Ask for feedback
</code></pre>

<p><img src="./readme_files/nested-list-rendered.png" alt="Rendered nested list"></p>

<h3>
<a id="task-lists" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#task-lists" aria-hidden="true"><span class="octicon octicon-link"></span></a>Task lists</h3>

<p>To create a task list, preface list items with <code>[ ]</code>. To mark a task as complete, use <code>[x]</code>.</p>

<pre><code>- [x] Finish my changes
- [ ] Push my commits to GitHub
- [ ] Open a pull request
</code></pre>

<p><img src="./readme_files/task-list-rendered.png" alt="Rendered task list"></p>

<p>If a task list item description begins with a parenthesis, you'll need to escape it with <code>\</code>:</p>

<p><code>- [ ] \(Optional) Open a followup issue</code></p>

<p>For more information, see "<a href="https://help.github.com/articles/about-task-lists">About task lists</a>."</p>

<h3>
<a id="mentioning-users-and-teams" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#mentioning-users-and-teams" aria-hidden="true"><span class="octicon octicon-link"></span></a>Mentioning users and teams</h3>

<p>You can mention a user or <a href="https://help.github.com/articles/setting-up-teams/">team</a> on GitHub by typing <code>@</code> plus their username or team name to trigger a <a href="https://help.github.com/articles/about-notifications">notification</a> and bring their attention to an issue or pull request. People will also receive a notification if you edit a comment to mention their username or team name.</p>

<p><code>@github/support What do you think about these updates?</code></p>

<p><img src="./readme_files/mention-rendered.png" alt="Rendered @mention"></p>

<p>Typing an <code>@</code> symbol will bring up a list of people or teams on a project. The list filters as you type, so once you find the name of the person or team you are looking for, you can use the arrow keys to select it and hit either tab or enter to complete the name. For teams, just enter the @organization/team-name and all members of that team will get subscribed to the issue.</p>

<p>The autocomplete results are restricted to repository collaborators and any other participants on the thread.</p>

<h3>
<a id="referencing-issues-and-pull-requests" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#referencing-issues-and-pull-requests" aria-hidden="true"><span class="octicon octicon-link"></span></a>Referencing issues and pull requests</h3>

<p>You can bring up a list of suggested Issues and Pull Requests within the repository by typing <code>#</code>. Type the issue or PR number or title to filter the list, then hit either tab or enter to complete the highlighted result.</p>

<p>For more information, see "<a href="https://help.github.com/articles/autolinked-references-and-urls">Autolinked references and URLs</a>."</p>

<h3>
<a id="using-emoji" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#using-emoji" aria-hidden="true"><span class="octicon octicon-link"></span></a>Using emoji</h3>

<p>You can add emoji to your writing by typing <code>:EMOJICODE:</code>.</p>

<p><code>@octocat :+1: This PR looks great - it's ready to merge! :shipit:</code></p>

<p><img src="./readme_files/emoji-rendered.png" alt="Rendered emoji"></p>

<p>Typing <code>:</code> will bring up a list of suggested emoji. The list will filter as you type, so once you find the emoji you're looking for, press <strong>Tab</strong> or <strong>Enter</strong> to complete the highlighted result.</p>

<p>For a full list of available emoji and codes, check out <a href="http://emoji-cheat-sheet.com/">emoji-cheat-sheet.com</a>.</p>

<h3>
<a id="paragraphs-and-line-breaks" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#paragraphs-and-line-breaks" aria-hidden="true"><span class="octicon octicon-link"></span></a>Paragraphs and line breaks</h3>

<p>You can create a new paragraph by leaving a blank line between lines of text.</p>

<h3>
<a id="ignoring-markdown-formatting" class="anchor" href="https://help.github.com/articles/basic-writing-and-formatting-syntax/#ignoring-markdown-formatting" aria-hidden="true"><span class="octicon octicon-link"></span></a>Ignoring Markdown formatting</h3>

<p>You can tell GitHub to ignore (or escape) Markdown formatting by using <code>\</code> before the Markdown character.</p>

<p><code>Let's rename \*our-new-project\* to \*our-old-project\*.</code></p>

<p><img src="./readme_files/escaped-character-rendered.png" alt="Rendered escaped character"></p>

<p>For more information, see Daring Fireball's "<a href="https://daringfireball.net/projects/markdown/syntax#backslash">Markdown Syntax</a>."</p>
